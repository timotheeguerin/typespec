## Default encoding of scalars

As in JSON, there is some [default handling](https://typespec.io/docs/libraries/http/encoding#bytes) of the common scalars like `utcDateTime`:

| Scalar Type      | Default Encoding  | Encoding name                           |
| ---------------- | ----------------- | --------------------------------------- |
| `utcDateTime`    | `xs:dateTime`     | `TypeSpec.Xml.Encoding.xmlDateTime`     |
| `offsetDateTime` | `xs:dateTime`     | `TypeSpec.Xml.Encoding.xmlDateTime`     |
| `plainDate`      | `xs:date`         | `TypeSpec.Xml.Encoding.xmlDate`         |
| `plainTime`      | `xs:time`         | `TypeSpec.Xml.Encoding.xmlTime`         |
| `duration`       | `xs:duration`     | `TypeSpec.Xml.Encoding.xmlDuration`     |
| `bytes`          | `xs:base64Binary` | `TypeSpec.Xml.Encoding.xmlBase64Binary` |

## Examples

### 1. Array of primitive types

#### Example 1.1 — Unwrapped primitive array

Each element uses the property name as its XML element name.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@encodedName("application/xml", "XmlPet")
model Pet {
  @Xml.unwrapped
  tags: string[];
}
```

</TabItem>
<TabItem label="XML">

```xml
<XmlPet>
  <tags>abc</tags>
  <tags>def</tags>
</XmlPet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      items:
        type: string
        xml:
          name: tags
  xml:
    name: "XmlPet"
```

</TabItem>
</Tabs>

#### Example 1.2 — Wrapped primitive array with custom wrapper name

The array is wrapped in a container element with a custom name.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@encodedName("application/xml", "XmlPet")
model Pet {
  @encodedName("application/xml", "ItemsTags")
  tags: string[];
}
```

</TabItem>
<TabItem label="XML">

```xml
<XmlPet>
  <ItemsTags>
    <string>abc</string>
    <string>def</string>
  </ItemsTags>
</XmlPet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      xml:
        name: "ItemsTags"
        wrapped: true
      items:
        type: string
        xml:
          name: string
  xml:
    name: "XmlPet"
```

</TabItem>
</Tabs>

### 2. Complex array types

#### Example 2.1 — Unwrapped array of models

Each model item uses its own XML name when unwrapped.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@encodedName("application/xml", "XmlPet")
model Pet {
  @Xml.unwrapped
  tags: Tag[];
}

@encodedName("application/xml", "XmlTag")
model Tag {
  name: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<XmlPet>
  <XmlTag>
    <name>string</name>
  </XmlTag>
</XmlPet>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Tag:
  type: "object"
  properties:
    name:
      type: "string"
  xml:
    name: "XmlTag"
Pet:
  type: "object"
  properties:
    tags:
      type: "array"
      items:
        allOf:
          - $ref: "#/definitions/Tag"
        xml:
          name: tags
  xml:
    name: "XmlPet"
```

</TabItem>
</Tabs>

### 3. Nested models

#### Example 3.1 — Basic nested model

A property referencing another model is serialized as a nested XML element using the property name.

<Tabs>
<TabItem label="TypeSpec">

```tsp
model Book {
  author: Author;
}

model Author {
  name: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<Book>
  <author>
    <name>string</name>
  </author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    author:
      $ref: "#/components/schemas/Author"
Author:
  type: object
  properties:
    name:
      type: string
```

</TabItem>
</Tabs>

#### Example 3.2 — Nested model with custom XML name on the model

When the referenced model has an `@encodedName`, the property name still takes precedence for the element name.

<Tabs>
<TabItem label="TypeSpec">

```tsp
model Book {
  author: Author;
}

@encodedName("application/xml", "XmlAuthor")
model Author {
  name: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<Book>
  <author>
    <name>string</name>
  </author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    author:
      allOf:
        - $ref: "#/components/schemas/Author"
      xml:
        name: "author" # Here we have to redefine this name otherwise in OpenAPI semantic the `XmlAuthor` name would be used
Author:
  type: object
  properties:
    name:
      type: string
  xml:
    name: "XmlAuthor"
```

</TabItem>
</Tabs>

#### Example 3.3 — Nested model with custom XML name on the property

Using `@encodedName` on the property overrides the element name for the nested model.

<Tabs>
<TabItem label="TypeSpec">

```tsp
model Book {
  @encodedName("application/xml", "xml-author")

}

model Author {
  name: string;
}
```

</TabItem>
<TabItem label="XML">

```xml

  <xml-author>
    <name>string</name>
  </xml-author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml

  type: object
  properties:
    author:
      allOf:
        - $ref: "#/components/schemas/Author"

        name: "xml-author"

  type: object

    name:
      type: string
```

</TabItem>
</Tabs>

### 4. Attributes

#### Example 4.1 — XML attributes and element name override

Properties marked with `@Xml.attribute` are serialized as XML attributes instead of child elements. `@Xml.name` overrides the element name.

<Tabs>
<TabItem label="TypeSpec">

```tsp
model Book {
  @Xml.attribute
  id: integer;

  @Xml.name("xml-title")
  title: string;

  author: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<Book id="0">
  <xml-title>string</xml-title>
  <author>string</author>
</Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
      xml:
        attribute: true
    title:
      type: string
      xml:
        name: "xml-title"
    author:
      type: string
```

</TabItem>
</Tabs>

### 5. Namespace and prefix (inline form)

#### Example 5.1 — Namespace on the model

Applying `@Xml.ns` to a model prefixes the root element with the namespace.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@Xml.ns("http://example.com/schema", "smp")
model Book {
  id: integer;
  title: string;
  author: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema">
  <id>0</id>
  <title>string</title>
  <author>string</author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
    author:
      type: string
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
```

</TabItem>
</Tabs>

#### Example 5.2 — Namespace on individual properties

Properties can declare their own namespace, which may differ from the model's namespace.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@Xml.ns("http://example.com/schema", "smp")
model Book {
  id: integer;

  @Xml.ns("http://example.com/schema", "smp")
  title: string;

  @Xml.ns("http://example.com/ns2", "ns2")
  author: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema" xmlns:sn2="http://example.com/ns2">
  <id>0</id>
  <smp:title>string</smp:title>
  <ns2:author>string</ns2:author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
      xml:
        prefix: "smp"
        namespace: "http://example.com/schema"
    author:
      type: string
      xml:
        prefix: "ns2"
        namespace: "http://example.com/ns2"
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
```

</TabItem>
</Tabs>

### 6. Namespace and prefix (normalized form)

#### Example 6.1 — Using an `@Xml.nsDeclarations` enum

Declare namespaces in an enum to avoid repeating URIs. Reference enum members with `@Xml.ns`.

<Tabs>
<TabItem label="TypeSpec">

```tsp
@Xml.nsDeclarations
enum Namespaces {
  smp: "http://example.com/schema",
}

@Xml.ns(Namespaces.smp)
model Book {
  id: integer;
  title: string;
  author: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema">
  <id>0</id>
  <title>string</title>
  <author>string</author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
    author:
      type: string
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
Namespaces:
  type: string
  enum:
    - http://example.com/schema
```

</TabItem>
</Tabs>

Multiple namespace declarations referenced on individual properties.

<Tabs>
<TabItem label="TypeSpec">

```tsp

@Xml.ns(Namespaces.smp)
model Book {
  id: integer;

  @Xml.ns(Namespaces.smp)
  title: string;

  @Xml.ns(Namespaces.ns2)
  author: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<smp:Book xmlns:smp="http://example.com/schema" xmlns:sn2="http://example.com/ns2">
  <id>0</id>
  <smp:title>string</smp:title>
  <ns2:author>string</ns2:author>
</smp:Book>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
Book:
  type: object
  properties:
    id:
      type: integer
    title:
      type: string
      xml:
        prefix: "smp"
        namespace: "http://example.com/schema"
    author:
      type: string
      xml:
        prefix: "ns2"
        namespace: "http://example.com/ns2"
  xml:
    prefix: "smp"
    namespace: "http://example.com/schema"
Namespaces:
  type: string
  enum:
    - http://example.com/schema
    - http://example.com/ns2
```

</TabItem>
</Tabs>

### 7. Property setting the text of the node

#### Example 7.1 — Text content with attributes

Using `@Xml.unwrapped` on a `string` property makes it the text content of the parent element. Combined with `@Xml.attribute`, you can model elements that have both attributes and text content.

<Tabs>
<TabItem label="TypeSpec">

```tsp
model BlobName {
  @Xml.attribute language: string;
  @Xml.unwrapped content: string;
}
```

</TabItem>
<TabItem label="XML">

```xml
<BlobName language="abc">
  ...content...
</BlobName>
```

</TabItem>
<TabItem label="OpenAPI3">

```yaml
BlobName:
  type: object
  properties:
    language:
      type: string
      xml:
        attribute: true
    content:
      type: string
```

</TabItem>
</Tabs>
